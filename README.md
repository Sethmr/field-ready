# FieldReady

**On-site fluency for WNC listing agents.** A spaced-repetition flashcard webapp built around the ~250 facts a Macon-County listing agent needs in muscle memory — septic permits, well GPM, log-home rot, FPE panels, NC forms, post-Helene insurance — so a buyer's husband can't catch her flat.

Live at **[getfieldready.app](https://getfieldready.app)**.

---

## What's in here

- **84 hand-curated cards × 11 variants each** (v0 original + 10 reworded). Variant picker selects randomly per session so the surface form stays fresh.
- **84 editorial illustrations** (`images/`), one per card, in a cohesive warm-earth palette.
- **FSRS scheduling** via `ts-fsrs` loaded from esm.sh.
- **PWA-installable** on iOS and Android (`manifest.json` + `icon.svg`). Add to Home Screen from Safari/Chrome → full-screen standalone app.
- **Three palettes** (Cream / Meadow / Dusk), soft-voice toggle (Got it / Not yet vs. Good / Again), reward intensity (Subtle / Playful / Loud), name field — all in Settings.
- **localStorage** for progress (FSRS state per card, append-only answer log, streak, wall). Per browser, per device. No backend, no account, never leaves the device.
- **No build step.** React + Babel Standalone in the browser. Just open `index.html`.

## Run locally

```bash
python3 -m http.server 8765 --directory .
# open http://localhost:8765
```

Or via the Claude Code preview config — `.claude/launch.json` exposes it as the `fieldready` server on port 8765.

## Architecture decisions worth knowing

- **No build step. Babel Standalone transforms JSX in the browser.** Slower first paint than a bundled app (~75KB of JSX + Babel), but the no-friction-deploy + edit-and-reload story is worth it for a personal-scope app.
- **Cards live in `cards-data.js` as classic-script window globals**, not ES modules. The HTML loads them via `<script src="cards-data.js">` before the JSX runs. The generation scripts read this file via a `Function` sandbox to extract the globals.
- **FSRS via esm.sh ES-module dynamic import** in a `<script type="module">` block that runs *before* Babel scripts. Babel rewrites `import()` calls so we can't kick it off from JSX directly — the host HTML kicks it off, the JSX consumes `window.__fsrsPromise`.
- **Variants are static, not runtime-generated.** Pre-generated by `scripts/generate-variants.mjs` (Anthropic Sonnet) and committed. Variant picker selects randomly at session-build time. Zero runtime AI cost. To regenerate, see `scripts/README.md`.
- **Images are pre-generated** by `scripts/generate-images.mjs` (Anthropic distills the visual subject, OpenAI gpt-image-1 draws it, ~$0.01 per image). Same reasoning as variants.
- **Mastery threshold = FSRS stability ≥ 60 days.** Aligned with the consolidation window where SRS research treats material as durably encoded into long-term memory. Tunable via `MASTERY_STABILITY_DAYS` in `state.jsx`.

## Adding cards or fixing variants

Hand-edit `cards-data.js` directly for one-off tweaks (e.g., a wrong number, a tone fix). For bulk regeneration of variants or images, run the scripts in `scripts/` — see `scripts/README.md`.

Re-running `generate-variants.mjs` OVERWRITES variants on targeted cards (v0 always preserved as the original). Scope with `--card <id>` or `--pack <name>` while iterating.

## Privacy

Every byte of progress data is in the user's browser `localStorage`. Nothing is uploaded, nothing is logged on a server. The only network requests after first load are to `esm.sh` (FSRS library, cached) and Google Fonts (cached). The app works offline after first load.
